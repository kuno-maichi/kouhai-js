# 第3話：「橋を架ける」vs「新しい道を作る」

「これが僕の研究の現状です」

　雅史先輩の研究発表を、私は研究室の後ろの席で聞いていた。週一回の研究室ゼミ。今日は先輩が自分の研究について発表する番だ。

　スクリーンには、複雑な図が映し出されている。

「ScratchからPythonへの段階的移行ツール。ブロック型の視覚的要素を残しながら、徐々にテキストコーディングに慣れさせる」

　先輩がレーザーポインターで図を指しながら説明する。

「まず第一段階では、Pythonコードをブロック状に表示。第二段階で、ブロックとテキストのハイブリッド。最終的に純粋なテキストコードへ...」

「質問」

　最前列に座っていた教授が手を挙げた。

「花咲君、これは結局、Scratchに似せたPythonを作っているだけではないか？」

　私の心臓がドキッとした。まさに私が思っていたことだ。

「いえ、これは橋渡しとして...」

「橋渡し、ね」

　教授の声には懐疑的な響きがあった。


◇◇◇


　発表が終わった後、研究室に戻った先輩は疲れた顔をしていた。

「お疲れ様でした」

　私はコーヒーを差し出した。

「ありがとう。厳しい質問が多かったな」

「でも、先輩の研究、すごく頑張ってると思います」

「うん...でも」

　先輩はコーヒーを一口飲んで、

「何か引っかかるものがあるんだ」

「引っかかる？」

「優美の言葉を聞いてから、ずっと考えてる。僕は本当に正しいアプローチをしているのか」

　私は先輩の隣に座った。

「つまり、Scratchに似せたPythonを作ってるんですか？」

　先輩が苦笑した。

「教授と同じことを...」

「違います」

　私は首を振った。

「私が言いたいのは...」

　どう説明すればいいだろう。私は自分の経験を思い出しながら話し始めた。

「小説でも同じことがあるんです。ライトノベルから純文学に移行させようとして、『純文学風ライトノベル』を書く人がいます」

「純文学風ライトノベル？」

「文体は硬いけど、内容はライトノベル。でも、それって誰も読まないんです」

　先輩が興味深そうに聞いている。

「なぜ？」

「だって、ライトノベルの良さも純文学の良さも、どちらも失ってるから」

　先輩の表情が変わった。何か気づいたようだ。

「それって、読者に媚びて本質を失った小説みたい」

「その通り！」

　私は身を乗り出した。

「本当に必要なのは、新しい体験を作ることでは？」


◇◇◇


「新しい体験...」

　先輩はホワイトボードの前に立った。今まで描いていた図を消して、新しく描き始める。

「今までの僕のアプローチは...」

　左側にScratch、右側にPython、その間に矢印。

「この間に『橋』を架けようとしていた」

「はい」

「でも、優美が言いたいのは...」

　先輩が図を描き直した。今度は、ScratchとPythonの下に、新しい円を描く。

「『新しい道』を作る？」

「そうです！」

　私は立ち上がった。

「Scratchの良さは『恐怖がない』こと。Pythonの良さは『表現力』。なら、両方の良さを持つ新しい何かを」

「でも、それは理想論じゃ...」

「理想論？」

　私は先輩の目を見つめた。

「昨日、先輩は言いました。『これは技術の問題じゃない、人の心の問題だ』って」

　先輩が黙った。

「心の問題なら、技術的な妥協じゃなく、心に寄り添う新しい解決策が必要です」

「...君の言う通りかもしれない」

　先輩が振り返った。

「でも、具体的にどうすれば？」


◇◇◇


「まず、既存の『橋渡し』アプローチの問題点を整理しましょう」

　私はノートを開いた。昨夜、考えをまとめておいたものだ。

「一つ目、見た目だけの類似性」

「見た目だけ？」

「ブロック風のPythonコードって、結局エラーは出るんですよね？」

「...出る」

「なら、恐怖の問題は解決してない」

　先輩が頷いた。

「二つ目、中途半端な制限」

「制限？」

「Scratchみたいに制限をかけたら、Pythonの表現力が失われる。でも制限しなかったら、結局複雑になる」

「確かに...ジレンマだ」

「三つ目、そして最も重要なのは...」

　私は深呼吸した。

「移行を前提としていること」

「え？」

「『いずれはPythonに』という前提が、プレッシャーになるんです」

　先輩が考え込んでいる。

「つまり...」

「はい。『君はまだ本物じゃない』というメッセージを常に送り続けることになる」

　それは、小説でも同じだ。「いずれは純文学を」なんて言われたら、今書いているものに自信が持てなくなる。


◇◇◇


「じゃあ、優美の考える『新しい道』とは？」

　先輩が真剣な表情で聞いてきた。

「それ自体で完結した、新しいプログラミング体験」

「完結した？」

「Pythonへの移行を前提としない。それ自体で十分に表現力があり、かつ恐怖を感じない」

　先輩が首を振った。

「それは矛盾してる。表現力と安全性はトレードオフ...」

「本当にそうでしょうか？」

　私は先輩の研究ノートを指差した。

「先輩の研究、『感情的反応』について調べ始めたんですよね？」

「ああ、君のおかげで」

「なら、エラーを『恐怖』じゃなく『対話』にできませんか？」

　先輩の手が止まった。

「対話...」

「小説でも、読者との対話が大切です。一方的に押し付けるんじゃなく、一緒に世界を作っていく」

　私は続けた。

「エラーメッセージが『あなたは間違っている』じゃなく、『一緒に考えましょう』だったら？」

「それは...」

　先輩の目が輝き始めた。研究者の好奇心に火がついた瞬間だ。

「面白い。エラーを敵じゃなく、パートナーにする」

「はい！」

「でも、技術的に可能か...」

「先輩」

　私は微笑んだ。

「不可能を可能にするのが、研究者の仕事では？」


◇◇◇


　その後、私たちは深夜まで議論を続けた。

「感情認識型エラーハンドリング...」

　先輩がホワイトボードに新しいコンセプトを書いていく。

「初回のエラーは特に優しく。でも、ただ優しいだけじゃない」

「どういうこと？」

「学習者の感情状態を推測して、それに応じた対応をする」

　私も興奮してきた。

「例えば？」

「短時間に同じエラーを繰り返したら、焦っている可能性が高い。その時は『深呼吸しましょう』から始める」

「それいい！」

「逆に、じっくり考えている様子なら、より詳しいヒントを」

　これだ。これが私たちの作る「新しい道」。

「先輩、これってもう、単なるプログラミング言語じゃないですね」

「うん。プログラミング学習の新しいパラダイムかもしれない」

　窓の外を見ると、もう日付が変わっていた。

「遅くなった。送っていくよ」

「大丈夫です。それより...」

　私は先輩の顔を見た。疲れているけど、目は輝いている。

「明日から、本格的に始めましょう。私たちの『新しい道』作りを」

「ああ。既存の枠にとらわれない、全く新しいアプローチを」

　私たちは拳を合わせた。

　橋を架けるのではなく、新しい道を作る。

　それは、プログラミング教育の常識への挑戦。でも、私の感じた恐怖を知っている今なら、きっとできる。

　技術と感情が融合した、今までにないプログラミング体験。

　その第一歩を、今夜踏み出したのだ。
