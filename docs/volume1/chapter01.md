# 第1話：「Scratchは楽しかったのに...」

「助けて！エラーが消えないの！」

　ベランダ越しに聞こえた悲鳴に、僕は手にしていた学会論文を置いた。隣の部屋に住む幼馴染、柳原優美の声だ。

「どうした？」

　カーテンを開けると、優美はノートパソコンを抱えて半泣きになっていた。

「Pythonが...Pythonが怖い...」

　画面を見せてもらうと、真っ赤なエラーメッセージが表示されている。

```
Traceback (most recent call last):
  File "novel_manager.py", line 3, in <module>
    print(character_name)
NameError: name 'character_name' is not defined
```

「これ、何か悪いことした？私、パソコン壊した？」

　優美の声は震えていた。

「いや、ただの変数定義忘れだよ」

「変数？定義？」

　彼女の目には恐怖が浮かんでいる。まるで怪物でも見たかのような表情だ。

「ちょっと待って。君、プログラミングやってたの？」

「Scratchなら！ほら、これ見て！」

　優美は別のタブを開いた。カラフルなブロックが組み合わさった画面。そこには立派な小説管理ツールができている。

「すごいじゃないか。キャラクター管理も、プロット分岐も実装してる」

「でしょ？Scratchだと楽しくて、3ヶ月で作れたの。それでもっと高度なことがしたくてPythonに挑戦したら...」

　彼女は画面の赤い文字を指差した。

「この赤い文字が出て、何もできなくなった」

「Scratchだとエラーなんて出なかったのに！」

　優美の言葉に、僕は苦笑した。

「それが問題なんだ」

「え？」

「Scratchで守られすぎて、現実のプログラミングに対応できない。これは教育研究でも大きな課題になってる」


◇◇◇


　僕の名前は花咲雅史。大学院で「プログラミング言語教育」を研究している。特に、ビジュアル言語からテキスト言語への移行問題が専門だ。

「僕の研究、まさにこの問題を扱ってるんだ」

　研究室に優美を連れてきて、ホワイトボードに図を描く。

「Scratchのようなビジュアル言語は、ブロックを組み合わせるだけ。構文エラーは原理的に起こらない」

「それっていいことでしょ？」

「初学者にはね。でも...」

　僕は別の図を描いた。

「実務や研究では、テキスト言語が必須。表現力も効率も段違いだから」

「じゃあ最初からPythonを教えればいいのに」

「それが上手くいかないんだ。多くの人が最初のエラーで挫折する」

　優美は自分の画面を見つめた。

「...私みたいに？」

「データによると、ビジュアル言語経験者の約7割が、テキスト言語への移行で躓く」

「7割も！」

「ただし、これは我々の大学での調査結果。一般化は難しいけど」

　研究者の悪い癖だ。つい限定的な言い回しをしてしまう。

「でも、なんで私、こんなに怖いの？」

　優美の問いに、僕は答えに詰まった。

「怖い？」

「うん。この赤い文字見ると、心臓がドキドキして、逃げ出したくなる」

　これは想定外だった。僕の研究では「理解度」や「修正時間」は測定していたが、「恐怖」という感情は...

「興味深い。その『怖さ』について、もう少し詳しく聞かせてもらえる？」


◇◇◇


「Scratchでプログラミングしてた時は、どんな気持ちだった？」

　研究室のソファに座った優美は、少し落ち着きを取り戻していた。

「楽しかった！ブロックをパズルみたいに組み合わせて、思った通りに動くと嬉しくて」

「エラーは？」

「エラーなんて出ないもん。間違った組み合わせはそもそもできないし」

　彼女はScratchの画面を見せてくれた。

「ほら、このキャラクター管理システム。登場人物の相関図も自動生成できるの」

「本格的だな」

「小説書いてると、設定の矛盾とか多くて。だからツール作ろうと思って」

　なるほど、明確な目的があったのか。

「それでPythonに移行しようと思った理由は？」

「Scratchだと限界があるから。ファイル保存とか、外部データの読み込みとか」

　優美の判断は正しい。でも...

「最初のエラーで、すべてが変わった？」

「...うん」

　彼女は画面の赤い文字を見つめた。

「この赤い文字、『ERROR』って文字、意味の分からない英語...全部が『お前はダメだ』って言ってるみたい」

「エラーメッセージは問題を指摘してるだけで...」

「先輩には分かるでしょうけど！」

　優美の声が大きくなった。

「私には呪文にしか見えない。Scratchの時みたいに、何が間違ってるか視覚的に分からない。ただ赤い文字で責められてるだけ」

　彼女の言葉が胸に刺さった。

　僕たち研究者は、エラーメッセージの「分かりやすさ」は研究してきた。でも「怖さ」は？感情的な拒絶反応は？

「優美、君の感じてる『恐怖』...それ、もっと詳しく教えてくれないか」

「え？」

「実は、その視点は既存研究にない。君の感覚が、新しいブレークスルーになるかもしれない」


◇◇◇


「つまり、先輩の研究でも、この『怖さ』は扱ってないの？」

「恥ずかしながら」

　僕は自分の研究資料を見せた。

「エラーメッセージの理解度調査、修正にかかる時間の統計、学習曲線の分析...でも、感情面は『今後の課題』で済ませてた」

「なんで？」

「定量化が難しいから。『怖い』をどう数値化する？」

　優美は少し考えて、

「小説の読者離脱率みたいに？」

「読者離脱率？」

「ネット小説だと、どのページで読者が離脱したか分かるの。で、離脱率が高いところには必ず理由がある」

　彼女はスマートフォンを取り出した。

「難しい設定説明、唐突な視点変更、理解できない専門用語...」

「それは...プログラミングのエラーと同じかも」

「でしょ？私、読者が『怖い』『難しい』と感じる瞬間が分かるの。長年の経験で」

　これは面白い。作家としての読者心理分析力。

「優美、提案があるんだけど」

「なに？」

「君の『恐怖』の感覚と、僕の技術的知識を組み合わせて、新しいアプローチを作れないかな」

「新しいアプローチ？」

「Scratchからテキスト言語への移行を、『怖くない』ものにする」

　優美の目が輝いた。

「それって...私みたいな人を救えるってこと？」

「理論上はね。ただし、簡単じゃない」

「やる！絶対やる！」

　彼女の反応の速さに驚いた。

「即決？」

「だって...」

　優美は自分のエラー画面を見つめた。

「この恐怖で、どれだけの人がプログラミングを諦めてるか。もったいないよ」

　その言葉に、昔の後輩を思い出した。優秀だったのに、最初のエラーで挫折した子。

「分かった。一緒に研究しよう」

「はい！でも先輩、一つ条件」

「なんだ？」

「この赤い文字、どうにかして。見るだけで心臓バクバクする」

　優美の率直な恐怖。これこそが、僕の研究に欠けていたピースかもしれない。

　ビジュアル言語とテキスト言語の架け橋。それは技術的な問題じゃない。人の心の問題だったんだ。
