# 第7話：条件分岐：選択の連続が未来を作る

　金曜日の放課後。優美とのプログラミングレッスンも今日で五日目だ。

　昨日のレッスンで、優美が作った変数のコードにドキッとしてしまった。『関係性：友達』から『関係性：大切な人』への変化。あれは本当にキャラクター設定だったのだろうか。

　優美の何気ない言葉や行動に、最近は過敏に反応してしまう自分がいる。

（俺、完全に優美のこと意識してるな）

　今日も優美が僕の部屋にやってきた。

「お疲れ様です、先輩！」

「おう、お疲れ。今日も頑張ろうな」

　優美の笑顔を見ると、心が軽くなる。

「今日は何を教えてくれるんですか？」

「今日は条件分岐について学ぼう。if文っていうやつだ」

　僕はキーボードに向かった。

「条件分岐？」

「簡単に言えば、『もしこうなら、こうする』っていう処理だ。優美の小説でも使ってるはずだよ」

「どういうことですか？」

「主人公が選択を迫られる場面って、よくあるだろ？『もし告白が成功すれば幸せになる、失敗すれば……』みたいな」

　優美の目が輝いた。

「あります！　主人公の行動によって物語の展開が変わる場面ですね！」

「そう。プログラミングでもそれと同じことができるんだ」

```javascript
let mood = "晴れ"

if (mood === "晴れ") {
    console.log("お散歩に行こう！")
} else if (mood === "雨") {
    console.log("家で読書をしよう")
} else {
    console.log("何をしようかな？")
}
```

「実行してみて」

　画面に「お散歩に行こう！」と表示された。

「すごい！　天気によって行動が変わるんですね！」

　優美が興奮している。

「そう。条件によって、プログラムの動作を変えることができるんだ」

◇◇◇◇

「今度は、優美の小説のシチュエーションで作ってみよう」

「私の小説で？」

「恋愛小説でよくある『告白シーン』を条件分岐で表現してみるんだ」

　僕は新しいコードを書き始めた。

```javascript
let confession = "成功"
let herResponse = "はい"

if (confession === "成功" && herResponse === "はい") {
    console.log("二人は恋人同士になりました")
} else if (confession === "成功" && herResponse === "少し待って") {
    console.log("彼女は考え中です")
} else {
    console.log("告白は失敗に終わりました")
}
```

「これを実行すると……」

　画面に「二人は恋人同士になりました」と表示された。

「わあ！　ハッピーエンドですね！」

　優美が手を叩いて喜ぶ。

「&&って何ですか？」

「『かつ』っていう意味だ。『告白が成功』かつ『返事がはい』の両方が成り立つ時だけ、恋人になれるんだ」

「なるほど！　両方の条件が揃わないといけないんですね」

　優美の理解が早い。

「じゃあ、優美も何か条件分岐を作ってみて」

「はい！」

　優美は考えながらキーボードを打ち始めた。

```javascript
let study = "プログラミング"
let partner = "先輩"

if (study === "プログラミング" && partner === "先輩") {
    console.log("毎日が楽しくて幸せです")
} else if (study === "プログラミング") {
    console.log("勉強は楽しいです")
} else {
    console.log("普通の日々です")
}
```

　実行すると、「毎日が楽しくて幸せです」と表示された。

　また優美が赤くなっている。

「これも……キャラクターの設定？」

　僕が聞くと、優美は慌てたように頷いた。

「そ、そうです！　キャラクターの心情を表現してみました！」

　でも、僕も実は嬉しかった。『先輩』という条件が含まれていることに。

◇◇◇◇

「条件分岐は、優美の小説の構成にも応用できるよ」

「どういうことですか？」

「読者の選択によって物語が分岐するインタラクティブ小説とかね」

```javascript
let readerChoice = "告白する"

if (readerChoice === "告白する") {
    console.log("第8章：勇気の告白")
} else if (readerChoice === "もう少し待つ") {
    console.log("第8章：静かな想い")
} else if (readerChoice === "諦める") {
    console.log("第8章：別れの決意")
}
```

「読者が選択できる小説！　面白そうです！」

「将来、優美がYume言語でそういう小説を書けるようになるかもしれないな」

　僕の言葉に、優美の表情が明るくなった。

「本当ですか？　私、そういう小説を書いてみたいです！」

「きっと書けるよ。優美なら」

　僕が微笑むと、優美も嬉しそうに微笑み返してくれた。

◇◇◇◇

「でも、条件分岐って、人生にもあてはまりますよね」

　優美がふと言った。

「どういう意味だ？」

「私たちも毎日選択をしてて、その選択によって未来が変わる」

　優美の言葉に、僕は考え込んだ。

「確かにそうだな。今日俺が優美にプログラミングを教えるって選択も、俺たちの未来を変えるかもしれない」

「そうですね。もし私がプログラミングに興味を持たなかったら、こうして一緒に勉強することもなかったし……」

　優美が少し寂しそうな表情を見せた。

「でも、優美は興味を持ってくれた。そして俺たちは一緒にいる」

「はい。とても幸せです」

　優美の素直な言葉に、僕の心が温かくなる。

「俺も……優美と一緒にいると幸せだよ」

　つい本音が出てしまった。

「本当ですか？」

「ああ。本当だ」

　お互い少し照れくさそうに微笑み合った。

◇◇◇◇

「あの、先輩」

「ん？」

「もし私がプログラミングが上達しなかったら、先輩は嫌になりませんか？」

　優美が不安そうに聞いてきた。

「そんなことあるわけないだろ」

「でも……」

「優美、プログラミングのif文で考えてみろよ」

　僕は新しいコードを書いた。

```javascript
let yumi_skill = "初心者"
let my_feeling = ""

if (yumi_skill === "上級者") {
    my_feeling = "尊敬して好き"
} else if (yumi_skill === "中級者") {
    my_feeling = "頼もしくて好き"
} else if (yumi_skill === "初心者") {
    my_feeling = "一緒に成長できて好き"
} else {
    my_feeling = "どんな優美でも好き"
}

console.log(my_feeling)
```

「実行してみて」

　画面に「一緒に成長できて好き」と表示された。

「先輩……」

　優美が感動したような顔をしている。

「条件がどれでも、結果は『好き』だろ？」

「はい……」

「優美がどんなにプログラミングが苦手でも、どんなに上達しても、俺の気持ちは変わらない」

　僕の言葉に、優美の目に涙がにじんだ。

「ありがとうございます……」

「泣くなよ。せっかく楽しくプログラミングしてるのに」

「嬉し涙です」

　優美が笑顔で涙を拭く。

◇◇◇◇

　レッスンが終わって、優美が帰る時間になった。

「今日も楽しかったです」

「俺もだよ。条件分岐、理解できたか？」

「はい！　選択によって未来が決まるって、プログラミングも人生も同じなんですね」

「そうだな。優美が俺に『プログラミング言語を作って』って頼んでくれた選択が、俺たちの今を作ってる」

「私の選択は正しかったでしょうか？」

「もちろんだ。最高の選択だったと思う」

　優美が嬉しそうに微笑む。

「明日も教えてください」

「もちろんだ。次はループ処理について学ぼう」

「楽しみです！」

　優美が手を振って帰っていく。

　一人になった部屋で、僕は今日のことを振り返った。

（俺も毎日選択をしてるな）

　優美の隣に座る時の距離。視線の向け方。言葉の選び方。

　すべてが僕の選択で、その選択が僕たちの関係を少しずつ変えている。

（明日はどんな選択をするんだろう）

　もう少し優美に近づこうか。それとも今のままでいようか。

　プログラミングのif文のように、僕の人生も条件分岐の連続だ。

　でも、一つだけ確実に言えることがある。

（優美と一緒にいる時間は、いつも最高の選択だ）

　そんな気持ちで、僕は明日への期待を膨らませた。

　条件分岐を学んだ今日、僕は改めて気づいた。

　すべての選択が僕たちの未来を作っている。そして、優美と出会えたことは、僕の人生で最高の条件分岐だったのかもしれない。
