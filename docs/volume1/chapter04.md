# 第4話：「プログラマーの日常」（優美視点）

「先輩、まだいたんですか」

　午後9時。研究室のドアを開けると、雅史先輩はモニターを3つ並べて何かに没頭していた。

「ああ、優美。君こそ遅いね」

「小説の締切が近くて。でも、先輩は？」

　私が近づくと、画面には複雑なコードがびっしり。

「ちょっと面白いことを思いついて」

　先輩の目が輝いている。研究者モードだ。

「見て。Pythonのエラーメッセージを解析するプログラム」

```python
def analyze_error_patterns(error_log):
    """初学者のエラーパターンを分析"""
    patterns = {
        'NameError': {'frequency': 0, 'recovery_time': []},
        'SyntaxError': {'frequency': 0, 'recovery_time': []},
        'TypeError': {'frequency': 0, 'recovery_time': []}
    }
```

「これで何が分かるんですか？」

「エラーの種類ごとに、初学者がどれくらい時間をかけて解決するか」

　先輩が別のウィンドウを開いた。グラフがずらりと並んでいる。

「見て。NameErrorは平均3.2分で解決。でもSyntaxErrorは...」

「8.7分？」

「そう。しかも興味深いのは」

　先輩がグラフを拡大した。

「同じSyntaxErrorでも、括弧の閉じ忘れは2分。でもインデントエラーは15分以上」

「どうして？」

「Pythonのインデントは視覚的に分かりづらい。特にScratchから来た人には」


◇◇◇


　私は先輩の横に座って、データを眺めた。

「先輩、これ全部一人で分析したんですか？」

「3ヶ月かけてね。被験者は学部生50人」

「すごい...」

　私が感心していると、先輩が照れくさそうに頭を掻いた。

「でも、君の『恐怖』という視点は入ってなかった」

「それは...」

「だから今、過去のデータを見直してる。エラーで諦めた人の行動パターンに共通点がないか」

　画面に新しいグラフが表示された。

「これ、何ですか？」

「エラー遭遇から諦めるまでの時間と、その時の行動」

　データを見て、私は息を呑んだ。

「みんな、5分以内に諦めてる...」

「そう。しかも見て」

　先輩がマウスでデータポイントを指した。

「諦めた人の8割が、エラーメッセージを最後まで読んでない」

「読んでない？」

「正確には、視線追跡データによると、赤い『Error』の文字で視線が止まってる」

　私の心臓がドキッとした。まさに私と同じだ。


◇◇◇


「優美、実は聞きたいことがあって」

　先輩が真剣な顔で振り返った。

「なんですか？」

「君の小説、全部読ませてもらった」

「え？」

　予想外の告白に顔が熱くなる。

「特に『銀河鉄道の向こう側』。プログラマーが主人公の作品」

　それは私が1年前に書いた作品だ。

「あれ、技術描写がすごく正確だった。誰かに取材した？」

「いえ、独学で...」

「独学？」

　先輩が驚いている。

「プログラミングに興味があったから、本を読んだりして」

「でも、実際にコードは書けなかった？」

「...はい。最初のエラーで、怖くなって」

　恥ずかしい過去だ。でも先輩は優しく微笑んだ。

「なのに、プログラマーの思考や感情は完璧に描写できてる」

「それは...」

「君には観察力と共感力がある。技術者の心理を外から理解できる稀有な才能だ」

　先輩の言葉に、胸が温かくなった。

「だから君の視点が必要なんだ」

　先輩はホワイトボードに向かった。

「僕たち技術者は、エラーに慣れすぎてる。でも君は違う」

「でも、それって弱点では...」

「違う」

　先輩がきっぱりと言った。

「君の『恐怖』は、何百万人もの初学者の代弁なんだ」


◇◇◇


「ところで先輩」

　私は思い切って聞いてみた。

「なんでプログラミング教育の研究を？」

　先輩の手が止まった。

「...妹がいるんだ」

「妹さん？」

「中学生。成績も良くて、理系科目が得意」

　先輩の表情が曇った。

「でも、プログラミングの授業で挫折した」

「それって...」

「最初のPythonでエラーが出て、泣きながら『お兄ちゃん、私バカなの？』って」

　胸が締め付けられた。

「違うって言ったけど、彼女はもう触ろうとしない」

「...」

「だから決めた。妹みたいな子を二度と作らないシステムを作るって」

　先輩の決意に満ちた横顔を見て、私は理解した。

　この人は、ただの研究者じゃない。

　誰かの痛みを知って、それを解決しようとしている。

　私と同じだ。

「先輩」

「ん？」

「絶対、作りましょう。恐怖を感じないプログラミング環境」

「ああ」

　先輩が微笑んだ。

「君となら、きっとできる」

　窓の外はもう真っ暗。でも研究室は温かい光に包まれていた。

　コーヒーを淹れに立った私は、ふと思った。

　この人と一緒なら、どんな困難も乗り越えられる気がする。

　技術と論理。

　感情と共感。

　互いに足りないものを補い合える、最高のパートナー。

　きっとそれは、研究だけじゃない。

　もっと大切な何かにも、つながっている気がして。

「優美、コーヒーに砂糖は？」

「ミルクだけでお願いします」

　日常の些細なやり取り。

　でもそれが、とても愛おしく感じられた夜だった。


◇◇◇


「あ、そうだ」

　コーヒーを持って戻ってきた私に、先輩が言った。

「明日、教授に新しい研究方針を報告するんだけど」

「はい」

「優美も同席してもらえる？共同研究者として」

　共同研究者。その響きに胸が高鳴る。

「私なんかが...」

「なんかじゃない」

　先輩は真剣な顔で私を見た。

「君の視点がなければ、この研究は成立しない。僕は技術は分かるけど、感情は分からない」

「でも、私はプログラミングができない」

「それがいいんだ」

　先輩は微笑んだ。

「できる人には見えない壁が、君には見える。それが貴重なんだ」

　そうか。私の弱点だと思っていたものが、実は強みになるんだ。

「実は僕も、昔はエラーが怖かった」

　先輩が視線を落とした。

「さっき言った妹の話、実は半分嘘」

「嘘？」

「妹は確かにいる。でも、プログラミングで挫折したのは...僕自身も、だった」

　意外な告白だった。

「中学の時、独学で始めたプログラミング。最初のエラーで1週間悩んだ」

「先輩も...」

「うん。でも諦めきれなくて、必死に調べて、やっと解決した時の喜び」

　先輩の目が遠くを見ている。

「その経験があるから、研究者になれた。でも同時に...」

「？」

「諦めた人の気持ちも、痛いほど分かる」

　だから妹が挫折した時、自分のことのように辛かったんだ。

「優美、君と出会えて良かった」

　突然の言葉に、顔が熱くなる。

「君は僕が忘れかけていた『恐怖』を思い出させてくれた。そして...」

　先輩が私の目を見つめた。

「それを乗り越える方法を、一緒に探してくれている」

　研究室の時計が10時を告げた。でも、時間なんてどうでもよかった。

　この人と一緒なら、どんな困難も乗り越えられる。

　技術的な壁も、感情的な恐怖も、全て。

　私たちは、最高のチームだ。
