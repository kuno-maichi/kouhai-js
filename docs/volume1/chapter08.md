# 第8話：「心理学との出会い」（優美視点）

「緊張しますね」

　心理学部の建物の前で、私は雅史先輩に言った。

「僕もだよ。心理学の先生に、プログラミングの話をするなんて」

　田中先生との面談が決まったのは昨日。学習心理学が専門で、特に「失敗に対する感情反応」を研究されているらしい。

「でも、優美が一緒だから心強い」

「え？」

「君の方が人の心を理解するのが得意だから」

　そう言われて、少し嬉しくなった。技術は分からないけど、人の気持ちなら...

「失礼します」

　研究室のドアをノックすると、思っていたより若い女性の先生が現れた。

「田中です。お忙しい中、ありがとうございます」

　40代前半くらい。穏やかな笑顔だけど、目が鋭い。さすが研究者だ。

「花咲です。こちら、共同研究者の柳原さん」

「よろしくお願いします」

　研究室は本でいっぱい。でも、プログラミングの本はない。当然だけど。


◇◇◇


「まず、お聞きしたいのですが」

　お茶をいただきながら、田中先生が口を開いた。

「なぜ、心理学にアプローチしようと？」

　雅史先輩が説明を始めた。エラーメッセージの改善から、個人差の発見まで。

「なるほど。興味深いですね」

　田中先生が頷いた。

「実は、私たちの分野では『学習性無力感』という概念があります」

「学習性無力感？」

「繰り返し失敗を経験することで、『自分には能力がない』と学習してしまう現象です」

　私はドキッとした。まさに私が感じていたことだ。

「元々は動物実験から発見されたのですが、人間の学習でも重要な要因です」

　田中先生がホワイトボードに図を描いた。

```
失敗 → 「自分が悪い」 → 無力感 → 学習回避
     ↓
  原因の帰属
```

「ポイントは『原因の帰属』。失敗を何のせいにするかで、その後の行動が変わります」

「帰属...」

「『自分の能力不足』と考えるか、『やり方の問題』と考えるかで、大きく違います」

　なるほど。同じエラーでも、「私がバカだから」と思うか、「やり方を変えれば解決する」と思うかで、その後の行動が変わる。


◇◇◇


「先生、質問があります」

　私は手を挙げた。

「どうぞ」

「なぜ人によって帰属のパターンが違うんですか？」

「良い質問ですね」

　田中先生の目が輝いた。

「主に3つの要因があります。『安定性』『統制可能性』『内在性』」

　また図を描いてくれた。

「安定性は『この失敗は一時的か、永続的か』。統制可能性は『自分でコントロールできるか』。内在性は『原因が自分の内部にあるか、外部にあるか』」

「具体的には？」

　雅史先輩が聞いた。

「例えば、テストで悪い点を取った時。『自分は数学が苦手だから』と考える人は、内在的で安定的な帰属をしています」

「それは良くない？」

「学習意欲を削ぎます。一方、『勉強方法が悪かった』と考える人は、統制可能で不安定な帰属。改善の余地があると感じるので、次も頑張れます」

　私は自分のことを思い出した。エラーが出た時、「私にはプログラミングの才能がない」と思った。これは内在的で安定的な帰属だ。

「つまり、エラーメッセージも、帰属のパターンを変えられる？」

　私の質問に、田中先生が頷いた。

「理論的には可能です。メッセージの文言で、『能力の問題』ではなく『手順の問題』として認識させる」

「どうやって？」

「例えば...」

　田中先生が考えながら言った。

「『エラーが発生しました』ではなく、『手順を確認しましょう』とか」

　雅史先輩が興奮した様子で言った。

「それ、僕たちが直感的にやっていたことかもしれません」


◇◇◇


「ただし」

　田中先生が少し表情を曇らせた。

「帰属パターンは、長年の経験で形成されています。一朝一夕には変わりません」

「そうですよね...」

　期待していただけに、ちょっとがっかりした。

「でも、希望もあります」

「どんな？」

「『成功体験』の積み重ねで、徐々に変えることは可能です」

　田中先生が新しい資料を見せてくれた。

「この研究では、小さな成功を重ねることで、学習性無力感を改善できることが示されています」

「小さな成功...」

「そうです。いきなり大きな課題ではなく、確実にできることから始める」

　これは使えそうだ。プログラミングでも、いきなり難しいプログラムではなく、簡単なものから。

「先生、もう一つ質問が」

　雅史先輩が言った。

「先ほどの個人差の話ですが、事前に判別する方法はありますか？」

「心理尺度を使えば、ある程度は可能です」

「心理尺度？」

「質問紙で、その人の帰属パターンや自己効力感を測定する方法です」

　田中先生がファイルを開いた。

「例えば、『困難な問題に直面した時、あなたはどう考えますか？』という質問に、選択肢で答えてもらう」

　具体的な例を見せてもらった。

A. 「自分には能力がないからダメだ」
B. 「やり方を変えれば解決できるはず」
C. 「運が悪かっただけだ」
D. 「問題が難しすぎる」

「Bを選ぶ人は、統制可能な帰属をする傾向があります」

「なるほど！」

　私は理解した。事前にアンケートで、その人の思考パターンを把握できれば、適切なメッセージを選択できる。


◇◇◇


「ただし、注意点があります」

　田中先生が真剣な顔になった。

「心理尺度は万能ではありません。人間の心は複雑で、状況によって変わります」

「どういうことですか？」

「普段は前向きな人でも、疲れている時や不安な時は、ネガティブな帰属をしがちです」

　確かに。私も、締切に追われている時は、すぐに「才能がない」と考えてしまう。

「つまり、固定的な分類ではなく、動的な対応が必要」

「動的？」

　雅史先輩が聞いた。

「リアルタイムで状況を判断して、その時その人に最適なメッセージを選ぶ」

「それは...技術的に可能でしょうか？」

　田中先生が困った顔をした。

「私には技術のことは分かりませんが...」

「僕たちが考えてみます」

　雅史先輩が引き取った。

「今日は貴重な話をありがとうございました」

「こちらこそ。とても興味深いプロジェクトですね」

　田中先生が微笑んだ。

「もしよろしければ、継続的にご相談させていただけませんか？」

「もちろんです。むしろ、私も学ばせていただきたい」

　研究室を出た後、雅史先輩と私は顔を見合わせた。

「すごい収穫だったね」

「はい。心理学、奥が深いです」

　技術と心理学の融合。

　想像していた以上に複雑だけど、それだけに可能性も大きい。

　私たちの研究は、また新しい地平を見つけた気がした。

「そういえば優美」

　学食に向かいながら、雅史先輩が言った。

「今日の話を聞いて、君のことがよく分かった」

「私のこと？」

「君は『統制可能な帰属』をする人だ」

　先輩が説明してくれた。

「最初のエラーで挫折したけど、『才能がない』じゃなくて『やり方が分からない』と考えていた。だから僕の説明で立ち直れた」

「そうかもしれません」

　確かに、完全に諦めたわけじゃなかった。どうすればいいか分からなくて困っていただけ。

「でも、完璧主義の面もある。だから『不完全』な状態が怖い」

「鋭いですね」

　先輩の分析に、自分でも納得した。

「田中先生の理論だと、君みたいなタイプには『段階的な成功体験』が効果的らしい」

「段階的？」

「いきなり完璧を目指さず、小さな成功を積み重ねる」

　なるほど。それで私の恐怖が和らいだのか。

「でも先輩、私たちの研究、本当に実用化できるんでしょうか？」

　ふと不安になった。心理学的理論は面白いけど、実際のシステムに組み込むのは大変そう。

「難しいけど、やってみる価値はある」

　先輩が自信を持って答えた。

「少なくとも、従来の『一律同じメッセージ』よりは改善できるはず」

「そうですね」

　私たちは新しい挑戦に向かって、確実に歩を進めている。

　技術と心理学の架け橋。

　それは、誰も歩いたことのない道だった。
